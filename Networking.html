> NETWORKING FUNDAMENTALS: IP ADDRESSING

The internet is simply a massive collection of computers sending electrical signals back and forth. Because computer hardware only understands the two states of ON (1) or OFF (0) (Binary), every piece of data, from a text message to a video file, must be represented in binary.

For these messages to reach the correct destination, every device on a network needs a unique identifier—its Internet Protocol (IP) Address. Just like data, this address must also be expressed entirely in binary.

Network ID vs. Host ID

An IP address has two critical functions rolled into one. It must tell the internet both:

  1. Which network the device belongs to.
  2. Which specific device is on that network.
The entire 32-bit binary IP address is split into these two logical sections: the Network ID and the Host ID.

Think of it like a mailing address:

  • The Network ID is the Street Name and Town. It routes the data to the correct local area.
  • The Host ID is the House Number. It identifies the one device (your PC or phone) that should receive the data.

Dividing your network

Imagine your primary network (192.168.0.0) is an entire Town. We use subnetting to divide this large area into smaller, separate neighbourhoods (sub-networks).

The key to subnetting is a simple trade-off: Every bit we borrow to create a new neighbourhood (Subnet) is a bit we lose for giving houses (Hosts) unique numbers.
As you adjust the slider below:

  • The Subnet Bits (gold) increase, which doubles the number of Subnets.
  • The Host Bits (grey) decrease, which halves the number of Usable Hosts in each subnet.
The golden rule is: The more neighbourhoods you create, the smaller each neighbourhood gets!

What the Numbers Mean:

  • Subnet Mask: The fence (`/n` value) that defines where one network segment ends and the next begins.
  • Host Bits Borrowed: The number of bits taken from the Host ID to create new subnets.
  • Total Subnets Available: The total number of new neighbourhoods you have created.
  • Usable Hosts Per Subnet: The count of devices (PCs/phones) that can fit in each subnet. (Total addresses minus 2 for the Network ID and Broadcast ID).
